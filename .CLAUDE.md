## 핵심 수식 — 구현 전 반드시 읽어라

### 왜 Sigmoid인가 (Softmax 절대 금지)

일반 cross-attention은 Softmax를 쓴다:
```
w[n] = exp(Q·K[n]) / sum_n exp(Q·K[n])   # zero-sum: 한 entity가 올라가면 다른 entity는 강제로 내려감
```

이러면 두 entity가 겹치는 픽셀에서 "한 entity의 attention이 높아지면 다른 entity는 0에 수렴"한다.
모델은 한 entity를 아예 렌더링하지 않는 방향으로 수렴 → **Disappearance(객체 소실)**.

VCA는 Sigmoid를 쓴다:
```
sigma[s, n, z] = sigmoid(Q[s] · K[n,z] / sqrt(D))   # 각 entity가 독립적으로 [0,1] 밀도를 가짐
```

sigma는 "이 픽셀에서 이 entity가 이 깊이에 존재할 확률"이다.
entity 간 경쟁이 없으므로 두 entity가 동시에 높은 밀도를 가질 수 있다.

### Ray-Marching Transmittance

NeRF의 volume rendering에서 가져온 개념.
깊이 z에 도달하는 빛의 비율 T[z]를 앞쪽 모든 entity의 밀도로 계산한다:

```
opacity[s, z]  = clamp(sum_n sigma[s, n, z], max=1.0)
T[s, z=0]      = 1.0                                        # 아무것도 가리지 않음
T[s, z=k]      = prod_{z'=0}^{k-1} (1 - opacity[s, z'])    # 앞 슬라이스들이 빛을 막음
weight[s, n, z] = T[s, z] * sigma[s, n, z]                 # 최종 attention weight
output[s]       = sum_{n,z} weight[s, n, z] * V[n, z]
```

**핵심 직관**: 앞 entity(z=0)의 sigma가 크면 T[z=1]이 작아진다.
뒤 entity(z=1)의 weight = T[z=1] * sigma[z=1]도 자동으로 작아진다.
외부 마스크 없이도 앞 entity가 뒤 entity를 물리적으로 가린다.

### cumprod 구현 시 off-by-one 주의

```python
# WRONG: T[z=0]이 1.0이 아님
T = torch.cumprod(1.0 - opacity, dim=-1)

# CORRECT: exclusive cumprod → T[z=0]=1, T[z=1]=1-opacity[0], T[z=2]=(1-opacity[0])*(1-opacity[1])
ones = torch.ones(*opacity.shape[:2], 1, device=opacity.device)
T = torch.cumprod(torch.cat([ones, 1.0 - opacity], dim=-1), dim=-1)[:, :, :-1]
```

```python
    if call.when == "call" and call.excinfo is not None:
        phase = _get_phase(item)

def pytest_runtest_setup(item):
    phase = _get_phase(item)
    if phase is None or phase == 0:
        return
    for prev in range(phase):
        if prev in _phase_failed:
            pytest.skip(f"Phase {prev} failed — skipping Phase {phase}")

def _get_phase(item):
    for m in item.iter_markers():
        if m.name.startswith("phase"):
            try:
                return int(m.name.replace("phase", ""))
            except ValueError:
                pass
    return None
```

## Phase 1 — `models/vca_attention.py` 구현 후 `harness/test_phase1_vca.py`

### 구현 (`models/vca_attention.py`)

**텐서 규약 — 절대 바꾸지 마라**
```
BF = B * F   (batch × frames)
S  = H * W   (spatial tokens)
D  = query_dim
N  = n_entities  (Phase 1: 2 고정)
Z  = z_bins      (Phase 1: 2 고정)
CD = context_dim (CLIP: 768)
```

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
from einops import rearrange
from typing import Optional


class LoRALinear(nn.Module):
    """base weight frozen, LoRA branch만 학습.
    초기화: lora_B=zeros → 초기 LoRA 기여 0 (원 논문 방식, 분산 안정)
    lora_A=randn*0.01은 의도적. kaiming으로 바꾸지 마라 — lora_B=zeros라 초기 출력은 어차피 0.
    """
    def __init__(self, in_f: int, out_f: int, rank: int = 8):
        super().__init__()
        self.weight = nn.Parameter(torch.empty(out_f, in_f), requires_grad=False)
        nn.init.kaiming_uniform_(self.weight)
        self.lora_A = nn.Parameter(torch.randn(rank, in_f) * 0.01)
        self.lora_B = nn.Parameter(torch.zeros(out_f, rank))

    def forward(self, x):
        return F.linear(x, self.weight) + F.linear(F.linear(x, self.lora_A), self.lora_B)


class VCALayer(nn.Module):
    """
    Volumetric Cross-Attention.
    핵심: Sigmoid (독립 밀도) + Transmittance (물리적 occlusion)
    절대로 Softmax로 바꾸지 마라 — zero-sum이 되어 Disappearance 발생

    context_dim 선택:
      SD 1.5  기반 AnimateDiff → context_dim=768
      SDXL    기반 (Dual CLIP) → context_dim=2048
    파이프라인을 확정하기 전까지 하드코딩하지 말고 반드시 인자로 받아라.
    """
    def __init__(self, query_dim, context_dim: int, n_heads=8,
                 n_entities=2, z_bins=2, lora_rank=8):
        # context_dim에 기본값 없음 — 호출부에서 명시적으로 지정 강제
        super().__init__()
        assert query_dim % n_heads == 0
        self.n_heads, self.n_entities, self.z_bins = n_heads, n_entities, z_bins
        self.head_dim = query_dim // n_heads
        self.scale = self.head_dim ** -0.5

        self.to_q = nn.Linear(query_dim, query_dim, bias=False)
        self.to_k = LoRALinear(context_dim, query_dim, rank=lora_rank)
        self.to_v = LoRALinear(context_dim, query_dim, rank=lora_rank)
        self.depth_pe = nn.Parameter(torch.randn(z_bins, context_dim) * 0.02)
        self.to_out = nn.Linear(query_dim, query_dim)

        self.last_sigma: Optional[torch.Tensor] = None         # (BF, S, N, Z)
        self.last_transmittance: Optional[torch.Tensor] = None # (BF, S, Z)

    def _expand_context(self, ctx):
        # ctx: (BF, N, CD) → (BF, N*Z, CD)
        pe = self.depth_pe.unsqueeze(0).unsqueeze(0)           # (1,1,Z,CD)
        ctx3d = ctx.unsqueeze(2) + pe                          # (BF,N,Z,CD)
        return rearrange(ctx3d, 'bf n z cd -> bf (n z) cd')

    def _transmittance(self, sigma):
        # sigma: (BF,S,N,Z) → T: (BF,S,Z)
        # T[z=0]=1, T[z=k]=prod_{z'<k}(1 - clamp(sum_n sigma[z'], max=1))
        opacity = sigma.sum(dim=2).clamp(max=1.0)
        ones = torch.ones(*opacity.shape[:2], 1, device=sigma.device, dtype=sigma.dtype)
        return torch.cumprod(torch.cat([ones, 1.0 - opacity], dim=-1), dim=-1)[:, :, :-1]

    def forward(self, x, context):
        # x: (BF,S,D)  context: (BF,N,CD)
        BF, S, _ = x.shape
        N, Z, h = self.n_entities, self.z_bins, self.n_heads

        Q = rearrange(self.to_q(x), 'bf s (h d) -> bf h s d', h=h)
        ctx3d = self._expand_context(context)
        K = rearrange(self.to_k(ctx3d), 'bf (n z) (h d) -> bf h (n z) d', h=h, n=N, z=Z)
        V = rearrange(self.to_v(ctx3d), 'bf (n z) (h d) -> bf h (n z) d', h=h, n=N, z=Z)

        scores = torch.einsum('bhsd,bhtd->bhst', Q, K) * self.scale
        sigma  = torch.sigmoid(rearrange(scores, 'bf h s (n z) -> bf h s n z', n=N, z=Z))

        sigma_mean = sigma.mean(dim=1)                         # (BF,S,N,Z)
        T = self._transmittance(sigma_mean)                    # (BF,S,Z)

        weights = T.unsqueeze(1).unsqueeze(3) * sigma          # (BF,h,S,N,Z)
        weights = rearrange(weights, 'bf h s n z -> bf h s (n z)')
        out = rearrange(torch.einsum('bhst,bhtd->bhsd', weights, V), 'bf h s d -> bf s (h d)')

        self.last_sigma = sigma_mean.detach()
        self.last_transmittance = T.detach()

        return x + self.to_out(out)
```

```python
    l(torch.randn(2, 64, 320), torch.randn(2, 2, 768)).mean().backward()
    assert l.to_k.lora_A.grad is not None
    assert l.depth_pe.grad    is not None

def test_base_weight_frozen():
    from models.vca_attention import VCALayer
    l = VCALayer(query_dim=320, context_dim=768, n_heads=8, n_entities=2, z_bins=2)
    l(torch.randn(2, 64, 320), torch.randn(2, 2, 768)).mean().backward()
    assert l.to_k.weight.grad is None

# Sigmoid vs Softmax 검증
def test_entities_independent():
    """
    FM-I4 반영: ctx = [+5, -5] 설정은 K anti-symmetric을 만들어
    sigmoid(x)+sigmoid(-x)=1 항등식으로 diff ≈ 0이 된다 — 잘못된 테스트.

    올바른 방법: ctx = [+5, +5] (동일 방향)
    → σ_entity0[s] = σ_entity1[s] (같은 점수)
    → 두 entity가 동시에 high인 비율 ≈ 50% (Sigmoid)
    → Softmax라면 zero-sum → both_high ≈ 0%
    """
    from models.vca_attention import VCALayer
    CD = 768
    l = VCALayer(query_dim=320, context_dim=CD, n_heads=8, n_entities=2, z_bins=2)
    ctx = torch.zeros(2, 2, CD)
    ctx[:, 0, :] = 5.0   # entity 0: large positive
    ctx[:, 1, :] = 5.0   # entity 1: same direction — 반대(±5)가 아님!
    l(torch.randn(2, 64, 320), ctx)
    s = l.last_sigma  # (BF, S, N, Z)
    both_high = ((s[:,:,0,:] > 0.5) & (s[:,:,1,:] > 0.5)).float().mean()
    assert both_high.item() > 0.1, \
        f"Sigmoid should allow both entities high simultaneously. " \
        f"both_high={both_high:.4f} (Softmax would give ~0)"
```

### 구현 (`toy/generate_toy_data.py`)

두 시나리오를 모두 구현한다.

**시나리오 A — Chain Links**
- ㄷ자 tube 두 개 (반원 arc + 직선 cylinder 조합)
- Link 0: XZ 평면, 빨강 / Link 1: YZ 평면, 파랑
- 각 link가 자신의 축으로 천천히 회전 → 프레임 7~9에서 교차 최대
- PyVista: `pv.Spline().tube()` + `pv.Line().tube()`

**시나리오 B — Robot Arms**
- 3-joint arm 두 개 (Cylinder + Sphere 조합)
- Arm 0: (-0.4,0,0) 기준 빨강 / Arm 1: (+0.4,0,0) 기준 파랑
- Forward kinematics로 shoulder/elbow 각도 시간에 따라 변화
- 두 forearm이 중앙에서 점점 더 깊게 교차

**Robot Arm Forward Kinematics — 좌표 계산 (이대로 구현하라)**

각 joint의 world position을 순서대로 계산한다.
회전행렬 R_y(θ)를 누적 적용하는 방식:

```python
def rot_y(deg):
    r = np.radians(deg)
    return np.array([[np.cos(r), 0, np.sin(r)],
                     [0,         1, 0        ],
                     [-np.sin(r),0, np.cos(r)]])

# Arm 0 기준 (base_pos = [-0.4, 0, 0])
# joint 길이: upper_arm=0.6, forearm=0.5

shoulder_world = base_pos + [0, 0, 0.1]          # base 높이
elbow_world    = shoulder_world + R_y(shoulder) @ [0, 0, 0.6]
wrist_world    = elbow_world    + R_y(shoulder + elbow) @ [0, 0, 0.5]

# 각 파트 mesh를 해당 위치로 translate
# upper_arm: shoulder_world → elbow_world 방향으로 cylinder
# forearm:   elbow_world   → wrist_world  방향으로 cylinder
# pv.Cylinder는 center와 direction으로 배치
```

Arm 1은 base_pos=[+0.4, 0, 0], 각도 부호 반전(shoulder=-arm0_shoulder, elbow=-arm0_elbow).
두 forearm이 프레임 7~9에서 중앙(0,0,z)에서 교차하도록 각도를 설정하라.

### `models/vca_processor.py` — attn_processor 어댑터

**hook 방식 금지. 반드시 unet.set_attn_processor() 공식 API 사용.**

```python
import torch
from typing import Optional
from models.vca_attention import VCALayer


class VCAAttnProcessor2_0:
    """
    VCALayer을 diffusers attn_processor 인터페이스로 감싸는 어댑터.

    사용법:
        vca = VCALayer(query_dim=320, context_dim=768, ...)
        proc = VCAAttnProcessor2_0(vca)
        processors = {
            k: proc for k in unet.attn_processors if 'attn2' in k  # cross-attn only
        }
        unet.set_attn_processor(processors)

    주의: encoder_hidden_states는 (BF, N, CD) 형태여야 함.
          표준 CLIP 77토큰 아님 — N entity 임베딩을 별도로 구성해서 넘겨라.
    """
    def __init__(self, vca_layer: VCALayer):
        self.vca_layer = vca_layer

    def __call__(
        self,
        attn,
        hidden_states: torch.Tensor,               # (BF, S, D)
        encoder_hidden_states: Optional[torch.Tensor] = None,  # (BF, N, CD)
        attention_mask: Optional[torch.Tensor] = None,
        **kwargs,
    ) -> torch.Tensor:
        if encoder_hidden_states is None:
            # self-attention fallback: scaled dot-product
            import torch.nn.functional as F
            from einops import rearrange
            BF, S, D = hidden_states.shape
            h = attn.heads
            scale = (D // h) ** -0.5
            q = rearrange(attn.to_q(hidden_states), 'b s (h d) -> b h s d', h=h)
            k = rearrange(attn.to_k(hidden_states), 'b s (h d) -> b h s d', h=h)
            v = rearrange(attn.to_v(hidden_states), 'b s (h d) -> b h s d', h=h)
            out = F.scaled_dot_product_attention(q, k, v, scale=scale)
            out = rearrange(out, 'b h s d -> b s (h d)')
            return attn.to_out[0](out)

        # cross-attention: VCALayer 사용
        return self.vca_layer(hidden_states, encoder_hidden_states)
```

### 손실 함수 (`models/losses.py`)

세 개의 손실. 각각이 독립적으로 테스트 가능해야 한다.

```python
def l_ortho(depth_pe: torch.Tensor) -> torch.Tensor:
    """depth_pe (Z, CD): z_bins 벡터들이 직교하도록. FM-T4 방지."""
    pe_norm = F.normalize(depth_pe, dim=-1)          # (Z, CD)
    gram = pe_norm @ pe_norm.T                        # (Z, Z)
    eye  = torch.eye(gram.shape[0], device=gram.device)
    return (gram - eye).pow(2).mean()

def l_depth_ranking(
    sigma: torch.Tensor,       # (BF, S, N, Z)
    depth_order: list[int],    # [front_entity_idx, back_entity_idx]
    margin: float = 0.05,
) -> torch.Tensor:
    """
    앞 entity(z=0)의 sigma가 뒤 entity보다 margin 이상 커야 함.
    depth_order = [0, 1] → entity 0이 앞, entity 1이 뒤.
    hinge loss: max(0, sigma_back[z=0] - sigma_front[z=0] + margin)
    """
    front, back = depth_order
    s_front = sigma[:, :, front, 0]   # (BF, S)
    s_back  = sigma[:, :, back,  0]   # (BF, S)
    return F.relu(s_back - s_front + margin).mean()

def l_diff(pred: torch.Tensor, target: torch.Tensor) -> torch.Tensor:
    """기본 diffusion 노이즈 복원 손실 (MSE)."""
    return F.mse_loss(pred, target)
```

## Phase 10 — 학습 유효성 검증 (Encoding Fix + sigma_consistency)

### 배경 — 발견된 문제 3가지 (Phase 10 착수 이유)

**문제 1: x가 의미 없는 dummy**
- 기존: `x[:, :3] = rgb_patches` → 3/64 차원만 채움 → x.std() ≈ 0.006
- Q = x @ W_Q 에서 Q ≈ 0 → sigmoid(0) = 0.5 고착
- **수정**: tile + normalize → x.std() = 0.30

**문제 2: ctx가 거의 동일**
- 기존: `ctx[ei, :3] = color, ctx[ei, 3] = depth` → 나머지 124차원 = 0
- 두 entity ctx 차이 평균 0.001 → K_e0 ≈ K_e1 → entity 분리 불가
- **수정**: entity one-hot(dims 4-5) + entity-specific sin/cos depth encoding → ctx_diff = 0.28

**문제 3: 학습 유효성 검증 없음**
- "loss finite" 만으로는 sigma가 depth 순서를 실제로 배우는지 확인 불가
- **수정**: sigma_consistency (front entity sigma > back entity sigma 비율) 측정

### 10-A: 인코딩 수정 (train_vca.py ToyVCADataset)

```python
# x: tile RGB → normalize to std=0.3
reps = (query_dim + 2) // 3
x = np.tile(x_raw, (1, reps))[:, :query_dim].astype(np.float32)
x = (x - x.mean()) / max(x.std(), 1e-6) * 0.3

# ctx: one-hot identity + entity-specific sinusoidal
ctx[ei, :3]  = mean_color
ctx[ei, 3]   = mean_depth / 10.0
ctx[ei, 4:4+N] = 0.0
ctx[ei, 4+ei]  = 1.0                              # one-hot identity
for k in range(4+N, context_dim):
    freq = (k - 4 - N + 1) * 0.5
    ctx[ei, k] = (0.3 * sin(depth*freq) if ei==0  # e0: sin
                  else 0.3 * cos(depth*freq))      # e1: cos → structurally different
```

**결과**: x.std()=0.300, ctx_diff=0.279

### 10-B: 학습 sanity check (`scripts/check_learning.py`)

```

AFTER  sigma_consistency=0.8750 sigma_separation=0.0090
LOSS_CURVE 0.0499 0.0487 ... 0.0465
LEARNING=OK
```

- **sigma_consistency**: 각 프레임에서 front entity sigma_z0 > back entity sigma_z0 비율
  - 0.5 = random, > 0.60 = LEARNING=OK
- **sigma_separation**: |mean(sigma_e0) - mean(sigma_e1)| 데이터셋 평균
- **LEARNING=OK 기준**: sigma_consistency > 0.60

**실험 결과** (epoch=10, lr=1e-3, chain scenario):
- BEFORE sigma_consistency=0.50 → AFTER sigma_consistency=0.875

## Phase 12 — AnimateDiff VCA 실제 파인튜닝

**목표**: Phase 11 파이프라인 위에서 VCALayer(LoRA + depth_pe)만 학습.
UNet은 frozen, toy/data 영상의 실제 mid_block hidden_states로 감독.

### Phase 10과의 차이

| | Phase 10 (toy PoC) | Phase 12 (실제 학습) |
|---|---|---|
| query_dim | 64 | 1280 (mid_block 실제 차원) |
| context_dim | 128 | 768 (CLIP 임베딩) |
| x 소스 | RGB 패치 dummy | UNet mid_block hidden_states |
| L_diff | 없음 | 있음 (표준 diffusion MSE) |
| UNet | 없음 | frozen (requires_grad=False) |

### 핵심 구현 원칙

- **FM-A1**: hook 금지 → `unet.set_attn_processor()` (Phase 11과 동일)
- **FM-A2**: UNet fp16, VCALayer fp32. 입출력만 dtype 변환
- **FM-I6**: loss 계산은 반드시 `last_sigma_raw` (grad 있음). `last_sigma`(detach)로는 gradient=0
- **UNet frozen 검증**: `assert sum(p.requires_grad for p in unet.parameters()) == 0`
- **gradient checkpointing**: `pipe.unet.enable_gradient_checkpointing()` (메모리 절약)

### 손실 구조

```
L_total = L_diff + 0.1 * L_depth_ranking + 0.05 * L_ortho

L_diff: F.mse_loss(pred_noise, noise)  -- autocast(fp16) 밖에서 fp32 계산
L_depth_ranking: hinge loss, sigma_raw[:,:,front,0] > sigma_raw[:,:,back,0]
L_ortho: ||G - I||_F²  (depth_pe 직교성, FM-T4 방지)
```

### TrainVCAProcessor (학습용)

```python
class TrainVCAProcessor:
    def __call__(self, attn, hidden_states, ...):
        x = hidden_states.float()        # fp16 → fp32
        vca_out = self.vca(x, ctx)       # sets last_sigma_raw (with grad)
        attn_out = vca_out - x           # 이중 잔차 방지
        return attn_out.to(hidden_states.dtype)
```

### Part B: generate_crossing.py

3가지 CrossingMode로 depth ordering 역전 보장:

| 모드 | 설명 | 역전 원리 |
|------|------|-----------|
| ORBIT | 두 mesh 공전 (r=0.6, 반대 방향) | 공전 각도에 따라 앞뒤 주기적 역전 |
| SQUEEZE | 정면 충돌 후 통과 | 중간 지점 통과 시 역전 |
| ROTATE | 제자리 Y축 회전 (반대 방향) | 회전 각도에 따라 역전 |

**compute_frame_transforms() 핵심**:
- ORBIT: trans = [r*cos(t), r*sin(t), 0], rot = degrees(t)
- SQUEEZE: trans = [-1.5 + 1.8t, 0, 0], 반대는 [+1.5 - 1.8t, 0, 0]
- ROTATE: trans 고정 (±0.3), rot = ±frame*(360/n_frames)

**SceneRenderer 재사용**: `from toy.generate_toy_data import SceneRenderer, CAMERAS`

## Phase 39: Entity-Slot Attention v2 — GT Visible-Weight Supervision

### 배경 및 Phase 38 문제점

Phase 38 (Entity-Slot Attention)은 올바른 방향이었으나 4가지 구조적 결함이 있었다.

| 결함 | 원인 | 증상 |
|------|------|------|
| RGB threshold metric | entity_score/chimera_rate → 배경색 오판, entity 소멸 미감지 | best.pt가 실제로 나빠질 때 선택됨 |
| scalar slot_blend | 전체 동일 강도 → collision 영역 / 단독 영역 구분 불가 | blend 학습이 무의미 |
| frozen q/k/v만 존재 | slot adapter 없음 → L_exclusive gradient 사실상 0 | slot이 F_global 복사에 그침 |
| l_diff 처음부터 세게 | slot_blend→0으로 수렴 압력 | "원본과 달라지지 마" 신호가 slot을 꺼버림 |

### Phase 39 핵심 수정 (4가지 필수)

1. **RGB metric 제거** → GT mask 기반 held-out validation
2. **GT visible-weight loss 추가** (`l_visible_weights`) — 가장 핵심
3. **slot0_adapter / slot1_adapter + per-pixel blend_head** 추가
4. **overlap-heavy held-out validation**으로 best checkpoint 선택 교체

### 새로운 손실 함수

```
L = λ_vis   × L_visible_weights      ← 핵심 (exclusive/overlap/bg별 GT target)
  + λ_wrong  × L_wrong_slot_suppression ← exclusive 영역 wrong entity weight 패널티
  + λ_depth  × L_depth                ← VCA sigma depth ordering
  + λ_ov     × L_overlap_ordering     ← 겹침 영역 front > back (보조)
  + λ_diff   × L_diff                 ← 생성 품질 (2-stage ramp-up)
```

**L_visible_weights** (핵심):
- exclusive e0: w0→1, w1→0
- exclusive e1: w0→0, w1→1
- overlap (front=e0): w0→1, w1→0
- background: loss 없음 (mask 없는 영역)

**2-stage training**:
- Stage 1 (60% epoch): λ_diff=0.0 → slot이 자유롭게 visible/depth/ordering만 학습
- Stage 2 (40% epoch): λ_diff 0→0.3 linear ramp-up → 품질 보존 점진 적용

### 아키텍처 변경

### Solo Reference 방식

solo renderer 없음 → Entity-prompted UNet forward 사용:
- entity_j ctx를 0으로 zeroed → UNet forward → `proc.last_blended` 저장 (no-grad)
- `blended0_ref`: entity1 ctx=0 상태의 composition output (entity0 dominant)
- `blended1_ref`: entity0 ctx=0 상태의 composition output (entity1 dominant)

**중요 버그 수정** (`last_blended` vs `last_F0`):
- `last_F0` (raw slot features)는 composite/solo forward에서 **동일** — masked-attn은

- 수정: `last_blended` = blend_map*(w0*F0+w1*F1+wbg*Fg) + (1-blend_map)*Fg
  → VCA sigma 차이(w1_solo≈0)가 반영되어 composite≠solo (non-zero loss)

## Phase 41: Spatial Localization Fix (Class-Balanced Sigma)

### 문제 진단 (Phase 40 root cause)

Phase 40 결과: iou_e0=0.083, iou_e1=0.101 — 80 epoch 학습 후에도 목표(0.30)와 큰 차이.

**근본 원인 (Phase 41에서 수정)**:

`l_sigma_spatial` 이전 버전:
```python
l0 = F.mse_loss(alpha0.float(), m0)  # uniform average over ALL 256 pixels
```

Entity coverage ≈ 4% (10/256 pixels). Uniform MSE:
- 96% background gradient → "alpha → 0" (매우 강함)
- 4% entity gradient → "alpha → 1" (매우 약함)
- 96:4 = 24:1 gradient imbalance → model learns alpha≈0 everywhere (MSE-optimal)
- → IoU stuck at 0.082-0.12 regardless of training epochs

**Phase 41 fix** (`models/entity_slot.py` 수정):

```python
def _balanced_mse(pred, gt):
    n_pos = gt.sum().clamp(min=1.0)
    n_neg = (1.0 - gt).sum().clamp(min=1.0)
    # entity pixels weight=1, bg weight = n_pos/n_neg → equal total gradient
    w = gt + (1.0 - gt) * (n_pos / n_neg)
    return ((pred.float() - gt).pow(2) * w).mean()
```

이로써 entity (4%) total gradient ≈ background (96%) total gradient.
Model이 alpha→0 shortcut을 택할 수 없게 됨.

**추가 수정**: `l_visible_weights` + background suppression (bg_weight=0.2)
기존 코드는 entity 픽셀만 감독 → background w0/w1이 자유롭게 상승.
Phase 41: bg 픽셀에서 w0,w1→0 penalty 추가.

### Phase 41 단순화

- Single stage (60 epochs) — S2/S3 solo distillation 제거
  Solo distillation은 id_margin 악화 + IoU 개선 없음 → Phase 42 이후 재검토
- lambda_sigma = 5.0 (from 2.0) — class-balanced loss 이후 더 강한 신호 허용
